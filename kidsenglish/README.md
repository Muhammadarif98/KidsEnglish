# KidsEnglish 🦖

Мультитенантная платформа для изучения английского языка детьми 6–10 лет.
Каждый преподаватель регистрирует свой аккаунт, создаёт контент и
получает **уникальный код класса**, по которому ученики заходят на его
уроки. Построено на vanilla HTML/CSS/JS + Firebase.

---

## 🚀 Локальный запуск (без Firebase — demo-режим)

ES-модули требуют http-сервер. Нельзя открывать через `file://`.

```bash
cd public
python3 -m http.server 8080
```

Откройте <http://localhost:8080>. В demo-режиме:

* **Ребёнок:** на главной используйте код `DEMO01` → увидите стандартный
  набор из 8 тем и 55 заданий (seed из `seed.js`).
* **Преподаватель:** кнопка "Я преподаватель" → любой email/пароль
  (локально не проверяется) → попадаете в админку демо-класса.
* Все данные живут в `localStorage` браузера.
* Сброс: `localStorage.clear()` в DevTools + F5.

## ☁ Подключение Firebase (production)

### 1. Firebase Config — уже заполнен

`public/assets/js/firebase-config.js` уже содержит ваши ключи проекта
`kidsenglish-51c58`. Если создадите другой проект или перенесёте —
просто замените блок `FIREBASE_CONFIG`.

### 2. Authentication

В консоли проекта (<https://console.firebase.google.com/project/kidsenglish-51c58/authentication/providers>):

* **Email/Password** → **Enabled** (должно быть уже сделано).
* **Google** → **Enabled** — рекомендуется, один клик для учителя.
* **Registration:** НЕ ставьте галочку *"Disable sign-up"* —
  это сломает публичную регистрацию преподавателей.
* Удалять вручную созданных юзеров не нужно — они просто будут работать
  как обычные преподаватели после первого входа (профиль создастся
  автоматически).

### 3. Firestore Database

Consol → **Firestore Database → Create database**:

* Режим: **Production** (важно! не Test mode, чтобы правила реально применились).
* Регион: ближайший к вашей аудитории (`eur3` для Европы).
* После создания → вкладка **Rules** → вставьте содержимое
  `firestore.rules` из корня репозитория → **Publish**.

### 4. Деплой через Firebase CLI

```bash
# Один раз
npm install -g firebase-tools
firebase login

# В корне проекта
cp .firebaserc.example .firebaserc
# (в .firebaserc замените "your-firebase-project-id" на "kidsenglish-51c58")

firebase deploy
```

После деплоя сайт доступен по адресу
**`https://kidsenglish-51c58.web.app`** — даёте эту ссылку первым
пользователям.

## 🗃 Схема данных (Firestore)

### `teachers/{uid}` — профиль преподавателя
```
{
  displayName: "Мария Ивановна",
  email:       "maria@example.com",
  classCode:   "MARIA24",           // уникальный, генерируется автоматически
  createdAt:   1713612345678
}
```

### `classCodes/{CODE}` — публичный справочник "код → владелец"
Нужен чтобы ребёнок по коду мог найти UID препода без авторизации.
```
{
  ownerId:     "abc123...",         // uid препода
  teacherName: "Мария Ивановна"     // для отображения детям
}
```

### `topics/{id}` — тема
```
{
  ownerId: "abc123...",            // uid владельца
  title:   "Животные",
  emoji:   "🐶",
  order:   1,
  enabled: true                    // false = скрыта, но не удалена
}
```

### `exercises/{id}` — задание
```
{
  ownerId:     "abc123...",
  topicId:     "abc12345_animals_k8j2",
  type:        "A" | "B",
  order:       1,
  question:    "Dog",                     // для A — англ. слово, для B — фраза с "___"
  options:     ["Кот","Собака","Птица","Рыба"],
  answer:      1,                         // индекс правильного в options
  explanation: "Dog — это собака 🐶",
  imageUrl:    ""                         // пусто → placeholder
}
```

### Безопасность

`firestore.rules` обеспечивает:

* Любой (даже не авторизованный) может **читать** темы, задания, профили
  преподов, справочник кодов — нужно для детей без регистрации.
* **Писать** в `topics`/`exercises` может только авторизованный юзер,
  причём только в документы, где `ownerId == request.auth.uid` —
  препод не может редактировать контент другого препода.
* Изменение поля `ownerId` в существующем документе запрещено
  (защита от угона).

## 🆕 Что автоматически происходит при регистрации препода

1. Firebase Auth создаёт запись в Authentication.
2. В `teachers/{uid}` создаётся профиль.
3. Генерируется уникальный `classCode` (6 символов, буквы+цифры без
   путаницы между O/0 и I/1). Пишется в `classCodes/{CODE}`.
4. В `topics` и `exercises` копируется seed: 8 тем + 55 заданий, чтобы
   новый препод увидел готовый контент, а не пустой экран. Он может
   удалить/редактировать/добавлять что хочет.

Всё в одной транзакции Firestore — если что-то упадёт, ничего не
останется в "полусостоянии".

## 🐞 Если что-то не работает

**«Пустая главная»** — запускаете через `file://`? ES-модули не грузятся.
Нужен http-сервер: `python3 -m http.server` в директории `public/`.

**«Такой код не найден»** — код чувствителен к регистру, хотя вводится
автоматически в верхнем регистре. Проверьте что в Firestore
коллекция `classCodes` не пустая.

**Препод зарегистрировался, но в админке темы не появились** — 
seed мог не скопироваться из-за ошибки с правилами. Откройте DevTools
Console, найдите ошибку Firebase, пришлите. Скорее всего правила не
опубликованы или есть конфликт. Выход — препод создаёт темы вручную
(seed-копирование одноразовое, не блокирует работу).

**«Permission denied: missing or insufficient permissions»** —
правила Firestore не опубликованы, или опубликованы в режиме test
mode (`allow read, write: if false;`). Обновите правила содержимым
`firestore.rules` и нажмите **Publish**.

**Google-вход ругается на popup-blocked** — сайт должен быть открыт в
обычной вкладке (не в iframe), и в браузере разрешены всплывающие окна
для этого домена.

## ✅ Соответствие ТЗ

ТЗ изначально был под single-tenant (раздел 4.1: "Нет публичной регистрации
— только ручное создание администратора"). Мы осознанно расширили
за рамки ТЗ до мультитенанта по запросу заказчика. Всё остальное —
дизайн/UX, типы заданий A и B, экран результата, CRUD админки,
адаптив 320/481/1025 — реализовано в полном объёме.

### Вне объёма (ТЗ, раздел 9)

* Регистрация учеников и личный кабинет (дети по-прежнему анонимные,
  заходят по коду класса без регистрации) — v2.0.
* Сохранение прогресса между сессиями — v2.0.
* Аудио-произношение — v2.0.
* Тип задания C (сопоставление) — v2.0.
* Аналитика и статистика в панели препода — v2.0.

### Осознанные упрощения

* Загрузка картинки — поле **URL** вместо drag-and-drop в Firebase
  Storage. Storage требует платного тарифа Blaze; когда подключите —
  добавлю upload-виджет отдельно.
* Нет верификации email преподавателей. Любой может
  зарегистрироваться. Если начнёт расти — добавим verify email,
  защиту от ботов (reCAPTCHA), модерацию. Это настраивается в Firebase
  Auth без правки кода.
